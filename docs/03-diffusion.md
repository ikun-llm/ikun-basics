# 第三章：Diffusion 扩散模型

> 从一团噪点里变出篮球

---

## 一句话版本

扩散模型的核心思路超简单：先把图片**一步步加噪声**变成纯噪点，然后训练一个网络**学会一步步去噪声**，生成时从纯噪点开始逐步去噪，就能"凭空"变出图片。

---

## 直觉理解

### 加噪声（前向过程）

想象你有一张篮球的照片 🏀：

```
第 0 步：清晰的篮球 🏀
第 1 步：稍微有点模糊
第 2 步：更模糊了
第 3 步：隐约能看到圆形
  ...
第 100 步：完全变成雪花噪点 📺

就像你把一杯清水里滴了一滴墨水：
→ 刚开始能看到墨水的形状
→ 慢慢扩散
→ 最终变成均匀的灰色
→ 这就是"扩散"名字的由来
```

### 去噪声（反向过程）

现在训练一个神经网络，教它**反着走**：

```
输入：有噪声的图片
输出：噪声稍微少一点的图片

第 100 步：纯噪点
第 99 步：噪点里隐约有东西
第 98 步：能看出一个圆形
  ...
第 1 步：挺清晰的篮球了
第 0 步：完美的篮球！🏀
```

### 生成新图片

训练好之后，想生成新图片？

```
1. 随机生成一团噪点（随机数）
2. 用训练好的网络去噪：100 → 99 → 98 → ... → 0
3. 得到一张全新的图片！

神奇的是：每次用不同的随机噪点开始，就能生成不同的图片
```

---

## 为什么这个方法有效？

```
关键洞察：

"破坏"很容易 — 加噪声谁都会，不需要学习
"修复"很难   — 但如果每次只修复一点点，就变得可学了

把一个困难的问题（凭空造图）拆成很多简单的小步骤（每步只去一点噪声）
```

用生活比喻：

```
直接让你雕刻一座雕像              → 太难了！
给你一块已经粗略成型的石头，再精修 → 容易多了
给你一块更粗略的石头，再粗修       → 更容易

扩散模型就是反复做"稍微精修一下"这个简单操作
从纯噪点开始，一步步精修出最终图片
```

---

## 加入文字控制：文生图

DALL-E、Midjourney、Stable Diffusion 是怎么做到"输入文字→生成图片"的？

```
在去噪的每一步，把文字描述也喂给网络：

"一只鸡在打篮球"
     ↓
去噪网络不仅要去噪，还要让图片符合文字描述

训练时：
  输入：加了噪声的"鸡打篮球"图片 + "鸡在打篮球"文字
  目标：去掉一层噪声

推理时：
  输入：纯噪点 + "一只鸡在打篮球"文字
  输出：一张鸡在打篮球的图片 🐔🏀
```

### 文字是怎么被理解的？

```
文字 → 文本编码器（CLIP）→ 文本向量 → 注入去噪网络

CLIP 是什么？
  一个同时理解图片和文字的模型
  它知道"篮球"对应什么视觉特征
  它知道"鸡"对应什么视觉特征
  所以它能指导去噪过程
```

---

## 关键概念速查

| 概念 | 解释 | 比喻 |
|------|------|------|
| 前向过程 | 给图片逐步加噪声 | 把清水变成墨水 |
| 反向过程 | 逐步去噪声还原图片 | 从墨水里提取清水 |
| U-Net | 去噪用的神经网络 | 负责"精修"的工匠 |
| 时间步 (timestep) | 当前在第几步 | 告诉工匠"现在该粗修还是精修" |
| 条件生成 | 加入文字/图片等条件 | 告诉工匠"我要一个篮球" |
| CLIP | 理解文字和图片关系的模型 | 翻译官，把文字翻译成视觉语言 |
| 采样器 | 控制去噪步骤的算法 | 工匠的工作计划表 |

## Diffusion vs GAN

在 Diffusion 之前，生成图片主要靠 GAN：

```
GAN（对抗生成网络）：
  两个网络打架：
  - 生成器：造假图片
  - 判别器：辨别真假
  像：造假币的 vs 验钞机，互相对抗变强

  优点：生成速度快（一步到位）
  缺点：训练不稳定，容易"模式坍缩"（只会画一种图）

Diffusion：
  一步步去噪
  优点：训练稳定，多样性好，质量高
  缺点：生成慢（需要跑很多步）

现状：Diffusion 基本赢了，DALL-E / Midjourney / Stable Diffusion 都用它
```

## 和大语言模型的关系

```
大语言模型（GPT/ikun-2.5B）：
  生成方式：一个 token 一个 token 按顺序生成
  生成目标：文字

Diffusion 模型：
  生成方式：从噪声逐步去噪
  生成目标：图片/视频/音频

共同点：
  都是"生成模型"——从无到有创造内容
  都需要大量数据训练
  都可以加条件控制（文字描述）

结合：
  文本用 LLM 理解
  图片用 Diffusion 生成
  → 多模态 AI（能聊天也能画画）
```

## 本章小结

```
扩散模型三步走：

1. 训练时：拿真图加噪，教网络去噪
2. 生成时：从随机噪点开始，一步步去噪
3. 加条件：文字/风格/图片→控制生成内容

核心思想：把"凭空创造"拆成很多步"微小修复"
```

> 如果 ikun-2.5B 是"说梗"的 AI，
> 那 Diffusion 模型就是"画梗"的 AI。
> 也许有一天我们能做 ikun-Art——输入"鸡你太美"，生成一幅画 🎨🐔
